The goal of the visual profile is to communicate ideas as clearly as possible, while not being restrictive. This is achived in combination with the Kalt — Stylebook.

The logo

The logotype is set in Helvetica Regular; Kalt. It is always be capitalized, even in URL's.

The logomark — also called the insignia — is the outline of one circle, inside the outline of two ovals. The outlines has to be 1 pixel on digital platforms, and as thin as possible in print, whilst still being visable. It is very forgiving and can be made in almost any graphic software, even just in text ( (o) ).

Font stack

Since not all operating systems have the same fonts available, we have decided to use a stack of fonts instead of a restrictive set of fonts. This means we support almost every system.

Display font

We decided to use a serif fonts for titles and headlines.

  1. Kalt Display
  2. STIXGeneral Regular
  3. Times New Roman 
$serif : "Kalt Display",
         "STIXGeneral Regular",
         "Times New Roman",
         Serif;

@mixin font-display {
  font-family: $serif;
  font-size: clamp(25px, 6.48vw, 50px);
  line-height: clamp(37px, 9.72vw, 75px);
  margin-bottom: clamp(12px, 3.24vw, 25px);
}

Body font

Used for large blocks of texts,

  1. Helvetica
  2. Arial
  3. Raleway
  4. Roboto 
$sans-serif: "Helvetica",
             "Arial",
             "Raleway",
             "Roboto",
             Sans-Serif;

@mixin font-body {
  font-size: clamp(14px, 3.6vw, 26px);
	font-weight: 300;
  line-height: 150%;
  font-family: $sans-serif;
}
p {
  @include font-body;
}

Monospace font

Used for code.

  1. Kalt Monospace
  2. Courier New 
$monospace: "Kalt Monospace",
             "Courier New",
             Monospace;

@mixin font-body {
  font-size: inherit;
	font-weight: 300;
  line-height: inherit;
  font-family: $monospace;
}
pre code {
  @include font-body;
}

Margins

The margins on both the web, in documents etc is a creative restriction that lets the content breathe. When you are restricted to using only a set amount of space you

Digital

The margins on digital surfaces should be equal on all sides, the width of the margin is determined by the width of the surface. The margin should be 7% of the width of the screen.

Exceptions.

  • The menu of a website or an app has only a 1px margin around it.
  • On ultrawide screens the width of the site will only reach 720px. Therefore, the margins on the left and right side will be larger.
